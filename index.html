Introduction

In Pokémon, every Pokémon (and many moves) belongs to a type. Types interact like a strategy system: some types naturally have an advantage over others. That advantage shows up as attacks doing more damage (super effective) or less damage (not very effective).

Classic triangle

  • Water beats Fire
  • Fire beats Grass
  • Grass beats Water

What is my app?

This project is a Vuforia-powered app that recognizes Pokémon Energy cards, locks in two scanned types, and automatically determines which type has the advantage, showing the winner instantly.

Goal

Make type matchups feel instant and visual using AR tracking.

Inputs

Two energy cards scanned one at a time, each saved to a slot.

How does it work?

  1. Scan card 1

    Present the first card to the camera. When Vuforia detects it, the UI shows a “Detected” message and the current type.

  2. Lock Slot 1

    Press Slot 1 to save that detected type.

  3. Scan card 2

    Repeat the same process for the second card.

  4. Lock Slot 2 → Determine winner

    Once Slot 2 is locked, the app compares the two types and shows the winner (or a tie).

UX detail

After Slot 2 is locked, the “Detected” message disappears and the winner text turns on.

My database

The project uses a Vuforia image target database containing all card images. Vuforia also gives each image a rating for how “readable” it is, and some images track better than others.

What’s inside

One image target per card, each mapped to a specific energy type.

Why ratings matter

Higher-rated images are easier to detect consistently and from more angles.

Key scripts

BattleUIManager.cs

Controls the battle flow and UI: saves types into Slot 1 and Slot 2, hides “Detected” after Slot 2, displays the winner, and resets.

CardType.cs

Defines the energy types (Fire, Water, Grass, etc.) so the rest of the code stays consistent and clean.

TypeChart.cs

Stores the matchup rules. When both slots are filled, it compares effectiveness and returns a winner or tie.

VuforiaCardReporter.cs

Runs on each ImageTarget. When tracking is found/lost, it reports the detected type to the UI manager.

VuforiaFocusSetup.cs

Forces autofocus when Vuforia starts, improving scan reliability.

ForceVFXRestart.cs

Restarts the lightning visual effect on a timer so it loops continuously.

Implementation issues

The biggest challenge was detection consistency. Some cards tracked better than others, and the app sometimes needed restarts to recover from missed detection.

Areas of improvement

UI polish

Better detection

More camera tuning, stronger target images, and improved feedback when tracking is weak.

Stability

Handle tracking loss more gracefully so the user doesn’t feel like they need to restart.
